将Hexo博客部署至CloudFlarePages

发表于 2023-02-06 更新于 2024-02-06 分类于 博客搭建
在我们编写博客时，如果是像我一样使用 Hexo 框架的话，那么我们可以使用静态页面托管。但是 GitHub 仓库的 GitHub Pages 要求必须把项目库公开。那么我今天来介绍如何将自己的 Hexo 博客部署在 CloudFlare Pages。

在我们编写博客时，如果是像我一样使用 Hexo 框架的话，那么我们可以使用静态页面托管。但是 GitHub 仓库的 GitHub Pages 要求必须把项目库公开。那么我今天来介绍如何将自己的 Hexo 博客部署在 CloudFlare Pages。

准备材料

  • GitHub / GitLab 账户
  • CloudFlare 账户

部署步骤

  1. 在 GitHub / GitLab 创建一个项目


  2. 本地创建博客并上传到GitHub

  3. 进入 CloudFlare 账户首页，转到“Pages”

  4. 点击“创建应用程序”→“连接到 Git”

  5. 添加账户选择自己的储存库

  6. 构建设置全部选择默认

  7. 构建完成后，你就可以使用 cf 生成的 pages.dev 二级域名来访问自己的博客了

  8. 可以在自定义域这里绑定自己的域名
